1Identify failed locator/action
Which locator or Playwright action failed?		Use Playwright MCP accessibility snapshot around the current page or failed state to compare the target role/name/text with the failed locator.
2Check locator text on visible GUI
Is the intended button/link/input text visible to the user?		Use MCP browser snapshot / accessibility tree to list visible roles and names. Prefer role+accessible-name over screenshot guessing.
3Check DOM/accessibility presence
Does the element exist in DOM/accessibility tree?		Use MCP snapshot and locator probe to confirm whether the target exists, is hidden, duplicated, or inside iframe/shadow DOM.
4Check actionability/interactability
If element exists, can it receive click/fill/select action?		Use MCP click/hover/fill probes only in diagnostic mode. Detect overlays, disabled state, offscreen location, detached DOM, and permission popups.
5Map fix to POM/reuse layer
Is the correct locator strategy implemented in pageObjects/pages/util layer?		Use observed role/name/testId from MCP plus RAG-indexed framework context. Patch pageObjects first, then page methods/helpers, not raw spec locators.
